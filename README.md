## 서론: 증명할 수 없지만 우회할 수 있는 벽

컴퓨터 과학에는 수십 년 동안 풀리지 않는 질문이 하나 있다. 어떤 문제들은 정말로, 영구적으로 어려운가 — 아니면 빠르게 검증할 수 있는 모든 문제는 마찬가지로 빠르게 풀 수도 있는가. 이것이 P vs NP 문제다.

세계 최고의 수학자들이 수십 년간 시도했지만, 어떤 NP 문제에서도 다항식 시간 해법을 찾지 못했다. 이 문제들은 서로 수학적으로 연결되어 있어서, 하나만 빠르게 풀면 나머지 전체가 자동으로 해결된다 — 그런데 단 하나의 실마리도 나오지 않았다. 수학자들은 이미 기존 도구들이 이 문제에 접근조차 할 수 없음을 증명했다. 증명은 없지만, 증거가 쌓일수록 확신만 강해지는 역설적 상황이다.

그런데 AI는 다른 방향을 선택했다. P = NP를 증명하는 것이 아니라, NP를 **보이지 않게** 만드는 것. 이 문서는 그 전략의 기술적 구조를 다룬다.

## P와 NP: 계산 복잡도의 구조

### 정의

| 클래스 | 정의 | 시간 복잡도 | 대표 예시 |
|--------|------|------------|----------|
| **P** | 다항식 시간 내 **풀 수** 있는 문제 | O(n^k) | 정렬, 최단경로, 행렬곱 |
| **NP** | 다항식 시간 내 **검증**할 수 있는 문제 | 검증 O(n^k), 탐색 O(2^n) 이상 | 스도쿠, SAT, 그래프 색칠 |
| **NP-hard** | 모든 NP 문제보다 어렵거나 동등한 문제 | O(n!) 이상 | 외판원 문제(TSP), 단백질 폴딩 |
| **NP-complete** | NP이면서 NP-hard인 문제 | — | 스도쿠, SAT, 해밀턴 경로 |
| **PSPACE** | 다항식 공간으로 풀 수 있는 문제 | NP보다 더 어려울 수 있음 | 바둑(Go) |

```
P ⊆ NP ⊆ PSPACE ⊆ EXPTIME

P = NP? → 미해결 (Clay Millennium Prize, 100만 달러 현상금)
현재 압도적 통념: P ≠ NP (수학적 증명은 없음)
```

### 검증과 탐색의 비대칭

NP의 핵심은 **검증(Verification)과 탐색(Search)의 폭발적 비대칭**이다.

```
스도쿠:
  검증: 완성된 9×9 퍼즐 확인 → O(n²)  (순식간)
  탐색: 빈 칸 채우기 → 최악 O(9^81) ≈ 10^77  (우주 수명보다 긴 시간)

외판원 문제 (30개 도시):
  검증: 주어진 경로의 총 거리 확인 → O(n)
  탐색: 최적 경로 완전 탐색 → O(n!) = 30! ≈ 2.65 × 10^32

단백질 폴딩:
  검증: 구조의 에너지 상태 계산 → 다항식 시간
  탐색: 가능한 3D 구조 공간 → O(2^n) 이상 (지수적)
```

이것이 NP의 본질이다. 답이 주어지면 순식간에 맞는지 알 수 있지만, 처음부터 답을 찾는 것은 탐색 공간이 폭발적으로 커진다.

## 왜 일상에서 NP 문제를 느끼지 못하는가

이론적으로 NP-hard인 문제들이 일상에서 마법처럼 풀리는 이유는 두 가지다.

### 이유 1: 최악의 경우 ≠ 실제 경우

NP 분류는 **의도적으로 구성된 최악의 경우(Worst Case)** 를 기준으로 한다. 실제 세계의 문제들은 구조를 가지고 있다.

```
이론적 TSP (최악): 도시들이 무작위로 배치
→ 완전 탐색 외 방법 없음

실제 배달 경로:
→ 도시는 지리적으로 클러스터를 이룸
→ 간선도로와 고속도로가 존재
→ 교통 패턴이 반복됨
→ 그리디 + 지역 최적화로 95% 이상의 해 빠르게 도달 가능

카카오맵의 서울 경로 탐색:
→ 이론: NP 영역의 그래프 탐색
→ 실제: 계층적 도로 구조 + A* 알고리즘 → 수십 밀리초
```

### 이유 2: 완벽한 답 대신 충분히 좋은 답

현실에서는 **최적해(Optimal)** 가 아니라 **근사해(Approximate)** 로 충분한 경우가 대부분이다.

```
배달 기사에게 필요한 것:
  ❌ 수학적으로 증명된 최적 경로 (계산 불가)
  ✅ 95% 최적 경로 (수초 내 계산 가능)

AlphaFold의 단백질 구조 예측:
  ❌ 에너지 최솟값의 완전 계산 (수십 년 소요)
  ✅ 알려진 구조 패턴에서 학습된 형태 추정 (수 분)

AlphaGo의 바둑:
  바둑판 경우의 수: 10^170 (우주 원자 수 초과)
  클래스: PSPACE-complete (NP보다 더 어려움)
  ❌ 완전 탐색: 구조적으로 불가능
  ✅ 수천만 기보 학습 + 자기 대국 → 직관 압축 → 4:1 승리 (이세돌, 2016)
```

AlphaGo의 승리는 단순한 바둑 게임이 아니었다. NP를 제거하는 것이 아니라 **학습된 직관으로 우회하는 전략이 실제로 작동한다**는 것을 전 인류 앞에 증명한 순간이었다.

## 트랜스포머: 언어를 NP에서 해방시킨 아키텍처

> **Attention Is All You Need** (Vaswani et al., 2017) — NIPS 2017  
> arXiv: https://arxiv.org/abs/1706.03762

언어 이해는 구조적으로 NP에 가까운 문제다. 문장 내 단어들의 관계를 완전히 파악하려면 이론적으로 모든 부분집합 조합을 고려해야 한다. 트랜스포머는 이것을 **학습된 행렬 연산** 으로 대체했다.

### RNN의 순차 처리 문제 (NP-like 구조)

```
문장: "나는 어제 친구와 함께 서울에서 맛있는 저녁을 먹었다"

RNN 방식:
  t=1: "나는" → h₁
  t=2: h₁ + "어제" → h₂
  t=3: h₂ + "친구와" → h₃
  ...
  t=9: h₈ + "먹었다" → h₉

문제점:
  ① 순차 의존성 O(n): 병렬화 불가, GPU 활용 불가
  ② Vanishing Gradient: 긴 문장에서 초기 정보 소실
  ③ "나는"↔"먹었다" 관계 파악 = 전체 체인 순회 필요
  ④ 번역 품질: 문장이 길수록 정보 손실로 열화
```

이것은 P 알고리즘으로 NP 구조의 문제를 풀려는 시도였다. 순차 처리의 병목은 해결되지 않았다.

### Self-Attention: 병렬 관계 계산

트랜스포머는 순차 처리를 완전히 버리고, 모든 단어 쌍의 관계를 **동시에** 계산한다.

**수식**:
```
Attention(Q, K, V) = softmax(QKᵀ / √d_k) · V

Q (Query):  "이 단어가 무엇을 찾는가?"
K (Key):    "이 단어는 무엇을 나타내는가?"
V (Value):  "실제로 전달할 정보"
```

**동작 원리**:
```
     나는  어제  친구와  서울에서  맛있는  저녁을  먹었다
나는 [0.9] [0.1]  [0.7]   [0.1]   [0.1]   [0.1]   [0.8]
어제 [0.1] [0.9]  [0.2]   [0.2]   [0.1]   [0.1]   [0.6]
...

→ 모든 단어 쌍 관계를 O(n²·d)에 동시 계산
→ "나는"↔"먹었다"의 관계: 체인 순회 없이 직접 연결
```

**Multi-Head Attention** — 여러 관점의 관계를 병렬 포착:
```
MultiHead(Q,K,V) = Concat(head₁, ..., head₈) · W^O
where head_i = Attention(Q·W_i^Q, K·W_i^K, V·W_i^V)

head₁: 문법적 관계 ("나는" → 주어)
head₂: 의미적 관계 ("먹었다" → 행위자)
head₃: 시제 관계 ("어제" → 과거형 동사)
...
```

### 복잡도 비교: 왜 이것이 NP 우회인가

| 방식 | 순차 연산 | 최대 경로 길이 | 병렬화 | 실제 계산 |
|------|----------|--------------|-------|---------|
| RNN | O(n) | O(n) | 불가 | 느림, 정보 손실 |
| CNN | O(1) | O(log n) | 가능 | 장거리 관계 약함 |
| **Self-Attention** | **O(1)** | **O(1)** | **완전** | **모든 쌍 직접 연결** |

```
NP적 접근 (완전 탐색):
  "문장에서 가장 중요한 관계를 모두 찾아라"
  → 모든 부분집합 조합: O(2^n) → 불가능

Transformer의 전략:
  학습된 W_Q, W_K, W_V로 관련성을 O(n²·d)에 근사
  → 완벽한 탐색 대신, 데이터에서 학습한 패턴으로 대체
  → 지수적 탐색 공간 → 다항식 시간 근사
```

**이것이 트랜스포머의 핵심**이다. 언어 이해라는 지수적 탐색 문제를 학습된 선형 변환으로 압축하여, NP-like 구조를 다항식 시간에 처리 가능한 형태로 만들었다. BERT, GPT, LLaMA 등 현대 LLM의 전부가 이 원리 위에 있다.

**성능 검증** (WMT 2014 기계번역):
```
모델                    EN-DE BLEU    학습 비용
ConvS2S (CNN)            25.16       1.5 × 10²⁰ FLOPs
Transformer (base)       27.3        3.3 × 10¹⁸ FLOPs  ← 46배 적은 비용
Transformer (big)        28.4        2.3 × 10¹⁹ FLOPs  ← SOTA 달성
```

## AI의 NP 우회 전략 전체 지형

트랜스포머가 연 문을 통해, 이후 AI 기술들은 각기 다른 NP 문제들을 우회하기 시작했다. 공통된 전략은 세 가지로 요약된다.

```
전략 A: 병렬 근사      — 지수적 탐색 → 행렬 연산으로 대체
전략 B: 저차원 압축    — 고차원 공간 → 저차원 구조로 근사
전략 C: 학습된 직관    — 완전 탐색 → 데이터 패턴으로 탐색 공간 사전 제거
```

---

### LoRA: 파인튜닝을 저차원 공간으로 압축

> **LoRA: Low-Rank Adaptation of Large Language Models** (Hu et al., 2021) — ICLR 2022  
> arXiv: https://arxiv.org/abs/2106.09685

**전략 B: 저차원 압축**

```
NP-like 상황:
  GPT-3(175B) 파인튜닝 = 175,000,000,000 차원의 파라미터 최적화
  → 고차원 비볼록(non-convex) 최적화 = 지수적 복잡도

LoRA의 통찰:
  "사전 학습된 모델이 새 태스크에 적응할 때
   실제로 필요한 가중치 변화(ΔW)는 저차원 구조를 가진다"

  W_updated = W_frozen + ΔW = W_frozen + B × A

  B ∈ ℝ^(d×r),  A ∈ ℝ^(r×k),  r ≪ min(d,k)

수치 예시 (d=4096, k=4096):
  Full Fine-tuning:  4096 × 4096 = 16.7M 파라미터
  LoRA (rank=4):     (4096×4) + (4×4096) = 32K 파라미터  → 500배 압축
```

| 지표 | LoRA | Full Fine-tuning |
|------|------|-----------------|
| 학습 가능한 파라미터 | GPT-3 대비 0.01% | 100% (175B) |
| GPU 메모리 | ~3배 감소 | 기준 |
| 추론 지연 | **0 (증가 없음)** | 기준 |
| 체크포인트 크기 | 10,000배 감소 | 기준 |

**NP 우회 원리**: 파라미터 공간 탐색을 전체 고차원에서 수행하는 대신, 실제 필요한 변화가 살고 있는 **저차원 부분공간(intrinsic dimension)** 만 탐색한다. 지수적 최적화 → 저차원 행렬 곱 문제로 축소.

### RAG: 지식 저장 문제를 외부 검색으로 우회

> **Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks** (Lewis et al., 2020) — NeurIPS 2020  
> arXiv: https://arxiv.org/abs/2005.11401

**전략 C: 학습된 직관 + 전략 A: 병렬 근사**

```
NP-like 상황:
  "모든 지식을 파라미터에 저장하고 필요할 때 꺼내라"
  → 지식량 ∝ 파라미터 수
  → 업데이트 = 전체 재학습 (수억 달러, 수개월)
  → 지식 검색 = 수십억 파라미터에서 패턴 탐색

RAG 아키텍처:
  Input Query
      ↓
  [Dense Retriever: DPR]
    Query → 768-dim 벡터
    FAISS 인덱스 (21M 문서) → Sub-linear 근사 최근접 탐색
      ↓
  Top-K 관련 문서 (밀리초 단위)
      ↓
  [Generator: BART/GPT]
    Concat(Query + Documents) → 자연스러운 답변 생성
      ↓
  p(y|x) = Σ p(z|x) · p(y|x,z)   [문서 k개에 대한 주변화]
```

**수치 성과**:
```
626M 파라미터 RAG > 11B 파라미터 T5 (closed-book)
  → 모델이 작아도 외부 지식 활용으로 더 높은 성능
```

**NP 우회 원리**: 지식 인코딩 문제(조합 폭발)를 파라미터에서 분리하고, FAISS 벡터 검색(O(log n) ~ O(√n))으로 외부화한다. 지식 업데이트 = 문서 인덱스 교체만으로 즉시 반영.

### Scaling Laws: 지수적 실험을 예측 가능한 법칙으로

> **Scaling Laws for Neural Language Models** (Kaplan et al., 2020) — OpenAI  
> arXiv: https://arxiv.org/abs/2001.08361

**전략 B: 저차원 압축** (탐색 공간 → 멱함수 관계로 압축)

```
NP-like 상황 (2020년 이전):
  "어떤 크기의 모델을, 어느 데이터로, 얼마나 학습해야 하는가?"
  → 하이퍼파라미터 탐색 공간: 조합 폭발
  → 대규모 실험마다 수백만~수천만 달러 소요
  → 결과 예측 불가

Scaling Laws의 발견:
  L(N) = (N_c / N)^0.076    (모델 크기)
  L(D) = (D_c / D)^0.095    (데이터 크기)
  L(C) = (C_c / C)^0.050    (연산량)

예측:
  파라미터 10배 증가 → 손실 16% 감소 (10^(-0.076) ≈ 0.839)
  데이터 10배 증가  → 손실 20% 감소 (10^(-0.095) ≈ 0.803)

최적 자원 배분 공식:
  N_opt ∝ C^0.73,  D_opt ∝ C^0.27
  → GPT-3(175B) 개발의 이론적 근거
```

**NP 우회 원리**: AI 개발의 하이퍼파라미터 탐색을 지수적 실험 공간에서 수행하는 대신, 멱법칙(Power Law)이라는 **단순한 함수 관계** 로 압축한다. 이로써 소규모 실험 결과로 대규모 결과를 예측 가능해졌다.

### DeepSeek-R1: 강화학습으로 추론 직관 창발

> **DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via RL** (DeepSeek-AI, 2025)  
> arXiv: https://arxiv.org/abs/2501.12948

**전략 C: 학습된 직관** (AlphaGo 자기 대국의 언어 추론 버전)

```
NP-like 상황:
  수학 증명, 코딩 문제 = 다단계 추론 트리 탐색
  → 각 단계마다 분기 → 지수적 상태 공간
  → 완전 탐색: 불가능

기존 접근:
  Pre-training → SFT (수십만 고품질 CoT 데이터) → RLHF
  → 데이터 생성 비용 극대화, 일반화 한계

DeepSeek-R1의 전략:
  Pre-training → GRPO (순수 강화학습만)

  GRPO 보상 설계:
    동일 문제 → 여러 답변 샘플링
    advantage_i = reward_i - mean(group_rewards)
    → Critic 모델 불필요 (계산 비용 50% 감소)

자발적 창발 과정:
  Phase 1 (0~2000 steps):  단순 답변
  Phase 2 (2000~6000 steps): 자기검증(self-verification) 출현
  Phase 3 (6000+ steps):    다중 접근 + 메타인지적 반성
```

**성능**:
```
벤치마크         DeepSeek-R1    OpenAI o1-1217
AIME 2024         79.8%           79.2%      ← 동등
MATH-500          97.3%           96.4%      ← 상회
Codeforces Elo    2029            1891       ← 상회

개발 비용: ~560만 달러 (GPT-4 대비 1/18 수준)
라이선스: MIT 오픈소스
```

**NP 우회 원리**: 다단계 추론 탐색 트리를 명시적으로 구성하는 대신, **결과 기반 보상만으로** 모델이 스스로 효율적인 휴리스틱을 발견하게 한다. AlphaGo가 자기 대국으로 바둑 직관을 내면화한 것과 동일한 원리의 언어 추론 버전이다.

### LLM Agents + MCP: 복잡 문제를 분해하여 협력 처리

> **The Rise and Potential of LLM-based Agents: A Survey** (Xi et al., 2023)  
> arXiv: https://arxiv.org/abs/2309.07864

> **Advancing Multi-Agent Systems Through Model Context Protocol** (Krishnan, 2025)  
> arXiv: https://arxiv.org/abs/2504.21030

**전략 A + C: 병렬 근사 + 학습된 직관의 분산 적용**

```
NP-hard 상황:
  "복잡한 소프트웨어를 설계·구현·테스트하라"
  → 전체 최적화 공간: PSPACE 이상
  → 단일 에이전트로 처리 불가

에이전트 분해 전략 (MCP 기반):
  전체 문제
      ↓
  [Planning Agent]     → 하위 목표 분해 (다항식 시간)
  [Coding Agent]       → 개별 모듈 구현 (다항식 시간)
  [Testing Agent]      → 검증 및 피드백 (다항식 시간)
  [MCP Context Layer]  → 에이전트 간 상태 공유

  N×M 통합 문제 해결:
    기존: N개 앱 × M개 데이터소스 = N×M 커스텀 커넥터
    MCP: N + M 개의 표준 구현만 필요
```

**MCP 실측 성과**:
```
크로스도메인 합성 정확도:  78% (단일 에이전트 53% 대비)
세션 간 컨텍스트 연속성:  83.7% (기존 42.3% 대비)
쿼리 응답 시간:           67% 단축
자동 처리율:              94% (에스컬레이션 없이)
```

**NP 우회 원리**: 전체 최적화 문제를 단일 에이전트가 처리하는 대신, 각 에이전트가 **다항식 시간에 처리 가능한 하위 문제** 만 담당한다. 분할 정복(Divide & Conquer)의 AI 에이전트 버전.

## 벽이 남아 있는 곳: 암호학

AI가 NP를 보이지 않게 만드는 방향으로 달려가는 동안, 한 영역은 반대로 NP의 벽 위에 **의도적으로** 서 있다.

지금 이 순간 인터넷 뱅킹, 메시지 앱, 온라인 결제를 보호하는 암호학은 NP hardness를 방패로 사용한다.

```
RSA 암호화의 기반:
  두 거대한 소수의 곱: p × q = N (공개키)
  복호화: N을 다시 p와 q로 인수분해

  예:
    17 × 19 = 323  →  323의 인수 찾기: 쉬움
    각각 512비트짜리 두 소수의 곱  →  현존 모든 컴퓨터 수백 년 소요

이것이 의도적 NP 활용이다:
  검증(암호화): O(n²)   → 순식간
  탐색(복호화): O(e^n)  → 현실적 불가능
```

만약 P = NP를 증명하는 알고리즘이 내일 발견된다면, 현재 인터넷 보안의 상당 부분이 하룻밤 사이에 무너진다. 이것은 기술의 약점이 아니다. 기술이 그 벽의 위치를 정확히 알고, 그것을 방패로 삼아 서 있는 것이다.

벽은 사라지지 않는다. **변형된다.** 대부분의 AI 응용에서는 경험 가능한 영역 너머로 밀려나고, 암호학에서는 의도적으로 전면에 내세워진다.

## 결론

AI 기술의 NP 문제 해결 전략을 정리하면:

```
2017  Transformer    순차 탐색(O(n!)) → Self-Attention 병렬 근사(O(n²·d))
2020  Scaling Laws   지수적 실험 공간 → 멱법칙 함수 관계로 압축
2020  RAG            완전 파라미터 기억 → FAISS 벡터 검색으로 외부화
2021  LoRA           고차원 파라미터 최적화 → 저차원 행렬 분해
2023  LLM Agents     전체 PSPACE 최적화 → 하위 문제 분해 및 협력
2025  DeepSeek-R1    명시적 추론 설계 → 강화학습으로 직관 창발
2025  MCP            단일 에이전트 병목 → 분산 컨텍스트 공유 시스템
```

이 모든 기술의 방향은 하나로 수렴한다.

**P = NP를 증명하는 것이 아니라, NP를 보이지 않게 만드는 것.**

수학적 사실과 경험 사이에는 언제나 간극이 있었다. 지구는 태양 주위를 돌지만 매일 아침 해가 뜨고, 빛의 속도는 유한하지만 스위치를 켜는 순간 방은 빛으로 가득 찬다. P ≠ NP는 수학적으로 참일 것이다. 그러나 트랜스포머가 언어를 이해하고, AlphaFold가 단백질 구조를 예측하고, AlphaGo가 신의 한 수를 두는 세계에서, 그 진리는 경험적으로 그 반대와 구별 불가능해지고 있다.

**볼 수 없는 한계는 기능적으로 존재하지 않는 한계와 같다.**

### 용어 정리

| 용어 | 설명 |
|------|------|
| **P** | 다항식 시간에 풀 수 있는 문제 클래스 |
| **NP** | 다항식 시간에 검증 가능한 문제 클래스 (탐색은 지수적) |
| **NP-complete** | NP이면서 NP-hard인 문제 (TSP, SAT, 스도쿠 등) |
| **PSPACE** | 다항식 공간으로 풀 수 있는 문제 클래스 (바둑 등) |
| **Approximation Algorithm** | 최적해 근사 비율이 보장된 알고리즘 |
| **Self-Attention** | 시퀀스 내 모든 위치 쌍 관계를 O(n²)에 계산하는 메커니즘 |
| **Low-Rank Approximation** | 고차원 행렬을 낮은 랭크 행렬 곱으로 근사하는 기법 |
| **Scaling Law** | 모델 크기·데이터·연산과 성능 간의 멱법칙 관계 |
| **GRPO** | Group Relative Policy Optimization — Critic 없는 RL 최적화 |
| **Intrinsic Dimension** | 고차원 데이터가 실제로 살고 있는 저차원 부분공간의 차원 |
